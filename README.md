# Claude Agents Experiment

A Python sandbox for building, testing, and documenting multi-agent patterns with the Anthropic SDK. The focus is on **token cost discipline** — every architectural decision is justified and measurable.

## What this project demonstrates

- Multi-agent orchestrator pattern (plan → dispatch → synthesise)
- Token cost optimisation: 85–90% reduction vs a naive Opus-everywhere baseline
- Self-documenting agents (`DocumentationAgent` maintains `documentation/` automatically)
- Token usage tracking with an interactive HTML dashboard
- Agentic tool-use loops with file read/write and test-running capabilities

## Agents

| Agent | Model | Role | Cost/call |
|---|---|---|---|
| `SimpleAgent` | Sonnet 4.6 | Single agent, multi-turn tool loop | ~$0.005 |
| `TeammateAgent` | Haiku 4.5 | Stateless single-turn worker | ~$0.0002 |
| `Orchestrator` | Sonnet 4.6 | Plans, dispatches workers, synthesises | ~$0.003–0.006 |
| `DocumentationAgent` | Haiku / Sonnet | Maintains docs, runs codebase audits | ~$0.004–0.05 |

## Token optimisation strategies

All implemented in `src/token_optimization.py`:

| Strategy | Mechanism | Savings vs Opus baseline |
|---|---|---|
| Model tiering | Haiku for workers, Sonnet for orchestrator | ~80% |
| Tight `max_tokens` | 512 workers / 1024 orchestrator | ~50–75% on output |
| Prompt caching | `cache_control` on system prompts | ~60–90% on input* |
| Context isolation | Workers get a brief, not full history | ~50–80% |
| `effort=medium` | Sonnet orchestrator only | ~10–20% |

\* Activates automatically once system prompts exceed the minimum (2K tokens for Sonnet, 4K for Haiku).

**Combined: ~85–90% cheaper than a naive Opus-everywhere approach.**

## Quick start

```bash
# 1. Clone and install
git clone https://github.com/piotrkroczak-ai/claude-agents-experiment.git
cd claude-agents-experiment
uv sync

# 2. Add your API key
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env

# 3. Run the multi-agent demo
uv run python examples/teammate_demo.py

# 4. Run unit tests (no API key needed)
uv run pytest tests/ -v -m "not live"

# 5. Run all tests including live API calls
uv run pytest tests/ -v

# 6. Generate the token usage dashboard
uv run python examples/usage_report.py
# → open documentation/token_report.html in your browser
```

## Token usage dashboard

Every API call is logged automatically to `documentation/token_usage.jsonl`. Run `usage_report.py` to generate an interactive HTML dashboard showing:

- Cumulative cost per run (line chart)
- Cost breakdown by agent (bar chart)
- Token mix by model (doughnut chart)
- Full call history table
